Central Leads Management System

A Laravel MVP for collecting form submissions from multiple websites through a central API and managing them in a Filament admin dashboard.

Overview

This project centralizes leads from different websites and forms into one dashboard. Each external website can submit leads through a secure API key, while the admin panel organizes companies, sites, forms, leads, logs, and pipeline stages.

Problem

Businesses with many websites often receive form submissions in separate inboxes or plugins. This causes duplicated work, missed leads, unclear sources, and weak reporting.

Solution

  • Central Laravel API endpoint for incoming website forms.
  • API key authentication using Bearer token or X-API-Key header.
  • Support for multiple companies, websites, and forms.
  • Filament admin dashboard with Arabic and English support.
  • Lead pipeline, notes, activities, API submission logs, and login attempt tracking.
  • Lead exporting and email testing for cPanel deployment.
  • Integration documentation for PHP, WordPress, and Laravel websites.

Integration Flow

Website Form→API Key→POST /api/leads
Validation→Lead Storage→Filament Dashboard
Notifications→Pipeline→Analytics / Export

My Role

  • Designed the MVP structure for multi-site lead collection.
  • Built Filament resources for companies, forms, leads, logs, and dashboard pages.
  • Implemented secure API submission using site API keys.
  • Prepared integration examples and cPanel deployment notes.
  • Added multilingual Arabic/English admin panel support.

Result

The system provides a practical foundation for lead tracking across multiple websites, replacing scattered inbox-based workflows with a central dashboard and secure integration layer.
